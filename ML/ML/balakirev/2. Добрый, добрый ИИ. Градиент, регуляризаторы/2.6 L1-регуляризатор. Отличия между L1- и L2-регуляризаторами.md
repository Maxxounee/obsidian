На предыдущем занятии мы с вами подробно познакомились с принципом L2-регуляризации коэффициентов модели и увидели, что эта эвристика действительно позволяет заметно уменьшить переобучение модели и увеличить ее обобщающие свойства. Но раз L2-регуляризатор – это всего лишь эвристика, то почему бы не попробовать и другие подходы, например, вместо квадрата нормы вектора весовых коэффициентов:

$$
\tilde{L}_{i}(\omega)=L_{i}(\omega)+\frac{\lambda}{2}||\omega_{i}||
$$
взять модули – норму первой степени:

$$
\tilde{L}_{i}(\omega)=L_{i}(\omega)+\frac{\lambda}{2}||\omega_{i}||=
L_{i}(\omega)+\lambda \cdot \sum^{n}_{i=1}|\omega_{i}|
$$
В машинном обучении это называется L1-регуляризацией. Чтобы ее применить на практике, например, в градиентных алгоритмах, нам нужно уметь вычислять производную по функции потерь. Но, как вы знаете из курса математики, функция `y=|x|` не имеет производной в точке `x=0`. Поэтому найти оптимальные коэффициенты аналитически у нас не получится. В этом случае применяют численные методы, например, градиентный спуск. Градиент функции потерь с L1-регуляризатором будет иметь вид:

$$
\tilde{L}_{i}(\omega)=\triangledown L_{i}(\omega) + \lambda \cdot sign(\omega)
$$

где
$$
\begin{align} \\
&&+1, \ x>0 \\
sign(x) = &&0, \ x=0 \\ 
&&-1, \ x<0 \\
\end{align}
$$

В итоге, стохастический градиентный алгоритм (SGD) можно записать, следующим образом:

$$
\omega^{(t+1)}=\omega^{(t)} - \eta \cdot \triangledown L_{i}(\omega) - \lambda 
\cdot sign(\omega)
$$
Формально, конечно, нужно было умножить последнее слагаемое на `eta`, но я просто переобозначил параметр `lambda=eta*lambda`.


Давайте посмотрим, как будет работать такой L1-регуляризатор в уже знакомой нам простой задаче классификации гусениц и божьих коровок. Изначально, у нас есть следующие результаты измерений:

![[ML/assets/ml/balakirev/2/59.png|center]]

Это обучающая выборка с двумерными входными векторами:

$$
x_{i}=[w_{i}, h_{i}]^{T}, \qquad i = 1,2,\dots,10
$$
и целевыми ответами:
$$
y_{i}\in{-1;+1}, \qquad i=1,2,\dots,10
$$
Давайте теперь расширим пространство признаков, но специально сделаем их линейно-зависимыми, например, такими:

$$
x_{i}=[w_{i}, h_{i}, 10w_{i}, 10h_{i}, 5 \cdot (w_{i} + h_{i})]^{T}, \qquad i = 1,2,\dots,10
$$
Видите, здесь последние три признака – это линейная комбинация первых двух. То есть, они не несут в себе никакой дополнительной информации и, фактически, не нужны для данной задачи бинарной классификации жуков.

Следующим шагом построим модель и определим функционал качества для обучения модели. Как всегда, выберем простой линейный классификатор вида:

$$
a(x)=sign(\langle\omega, x \rangle)
$$

который выдает -1 гусениц и +1 – для божьих коровок. А эмпирический риск:

$$
Q(a, X^{l}) = \sum^{l}_{i=1}[a(x_{i}) \neq y_{i}] \leq \sum^{l}_{i=1}L_{i}(\omega, x_{i})

$$

будем минимизировать уже знакомой нам сигмоидной функцией потерь:

$$
L(\omega, x_{i}, y_{i})=\frac{2}{1+e^{M}}=\frac{2}{1+e^{\omega^{T}\cdot x_{i}\cdot y_{i}}}
$$

с производной по `w`, равной:

$$
\frac{dL(\omega, x_{i}, y_{i})}{d\omega}=-\frac{2}{(1+e^{\omega^{T}\cdot x_{i}\cdot y_{i}})^{2}}
\cdot
e^{\omega^{T}\cdot x_{i}\cdot y_{i}} \cdot x^{T}_{i} \cdot y_{i}
$$
Такую задачу мы с вами уже решали. Добавим сюда L1-регуляризатор и стохастический градиентный алгоритм примет вид:

$$
\omega^{(t+1)}=\omega^{(t)}-\eta \ \cdot \frac{dL(\omega, x_{i}, y_{i})}{d\omega}
-
\lambda \cdot sign(\omega)
$$

```python
import sys  
  
import numpy as np  
import matplotlib.pyplot as plt  
  
# исходная функция, которую нужно аппроксимировать моделью a(x)  
def func(x):  
    return -0.5 * x ** 2 + 0.1 * x ** 3 + np.cos(3 * x) + 7  
  
  
# модель  
def model(w, x):  
    xv = np.array([x ** n for n in range(len(w))])  
    return w.T @ xv  
  
  
# функция потерь  
def loss(w, x, y):  
    return (model(w, x) - y) ** 2  
  
  
# производная функции потерь  
def dL(w, x, y):  
    xv = np.array([x ** n for n in range(len(w))])  
    return 2 * (model(w, x) - y) * xv  
  
  
coord_x = np.arange(-4.0, 6.0, 0.1)  
coord_y = func(coord_x)  
  
  
  
N = 5 # сложность модели (полином степени N-1)  
lm_l1 = 2.0 # коэффициент лямбда для L1-регуляризатора  
sz = len(coord_x)  # количество значений функций (точек)  
eta = np.array([0.1, 0.01, 0.001, 0.0001, 0.000002]) # шаг обучения для каждого параметра w0, w1, w2, w3, w4  
  
w = np.zeros(N) # начальные нулевые значения параметров модели  
w_l1 = np.zeros(N) # начальные нулевые значения параметров модели  
w_l2 = np.zeros(N) # начальные нулевые значения параметров модели  
  
n_iter = 500 # число итераций алгоритма SGD  
lm = 0.02 # значение параметра лямбда для вычисления скользящего экспоненциального среднего  
batch_size = 20 # размер мини-батча (величина K = 20)  
  
Qe = 0 # начальное значение среднего эмпирического риска  
np.random.seed(0) # генерация одинаковых последовательностей псевдослучайных чисел  
  
# L2  
X = np.array([[x ** n for n in range(N)] for x in coord_x])  
IL = lm * np.eye(N)  
IL[0][0] = 0  
A = np.linalg.inv(X.T @ X + IL)  
w_l2 = A @ X.T @ np.array(coord_y)  
  
for _ in range(n_iter):  
    k = np.random.randint(0, sz - batch_size - 1)  # sz - размер выборки (массива coord_x)  
    batch_x = coord_x[k:k+batch_size]  
    batch_y = coord_y[k:k+batch_size]  
  
    # Без регуляризатора  
    dqdw = np.mean([dL(w, x, y) for x, y in zip(batch_x, batch_y)], axis=0)  
    w = w - eta * dqdw  
  
    # L1  
    Qk = np.average([loss(w_l1, x, y) for x, y in zip(batch_x, batch_y)])  
    Qe = lm * Qk + (1 - lm) * Qe  
  
    wt_l1 = w_l1.copy()  
    wt_l1[0] = 0  
    dqdw_l1 = np.mean([dL(w_l1, x, y) for x, y in zip(batch_x, batch_y)], axis=0)  
    w_l1 = w_l1 - eta * (dqdw_l1 + lm_l1 * np.sign(wt_l1))  
  
  
  
Q = np.average([(model(w_l1, x) - func(x)) ** 2 for x in coord_x])  
  
# График  
  
# sys.exit()  
  
plt.plot(coord_x, coord_y)  
plt.plot(coord_x, model(w_l1, coord_x))  
plt.plot(coord_x, model(w_l2, coord_x))  
plt.plot(coord_x, model(w, coord_x))  
plt.show()
```
